# Frontend

## Main Screens
- `Home.jsx`: 가장 큰 화면. 할일, 습관, 챌린지, 인생 목표, 반복 할일, 홈 구성 설정을 포함한다.
- `History.jsx`: 달력/검색/주간 스케줄 진입점.
- `Today.jsx`: 오늘 일기/메모 진입점.
- `Community.jsx`: 커뮤니티/챌린지 홈과 상세 흐름.
- `Settings.jsx`: 환경설정, 친구 초대, 각종 서브페이지.

## Community Notes
- 공개/초대코드 가입 시 비밀번호가 있는 커뮤니티는 가입 단계에서 암호를 확인한다.
- 이미 가입된 커뮤니티도 상세 진입하거나 다른 커뮤니티로 전환할 때 비밀번호가 있으면 매번 다시 묻는다.
- 현재 재입력 UI는 커스텀 모달이 아니라 브라우저 `window.prompt` 기반이다.

## Home Notes
- 홈 섹션 순서는 `dm_home_section_order`에 저장한다.
- 홈 표시 토글은 `dm_home_prefs`에 저장한다.
- 최근 홈 설정 UI는 `src/components/home/`로 일부 분리했다.
- 홈 구성 팝업은 현재 드래그를 포기하고 토글 + 위/아래 화살표 정렬 방식으로 운영한다.
- 홈 섹션 기본 순서는 `config.js` 기준으로 보정되며, 예전 저장값에 없는 섹션(`level` 등)은 기본 위치에 끼워 넣는다.
- 홈 파일은 여전히 크므로 추가 기능은 가능하면 컴포넌트 단위로 분리한다.

## Goals Notes
- 목표 데이터는 `year`, `month`, `months` 구조를 함께 다루며, 실제 월별 분리는 `months[YYYY-MM]`를 기준으로 한다.
- `raw.month`는 레거시 호환용이며, structured `months`가 있는 경우에는 월 전환 시 fallback으로 덮어쓰면 안 된다.
- 최근 `3월/4월이 같은 목표로 보이는 현상`은 이 fallback 조건 문제였고 로컬 코드에서 수정됨. 아직 배포 여부는 `update-log.md`와 최종 배포 기록을 함께 확인한다.

## Drag & Ordering
- 습관, 커뮤니티, 홈 섹션 순서 변경은 `@dnd-kit` 기반이다.
- 습관 정렬은 손가락 위치와 어긋남 문제를 피하려고 오버레이 없이 원본 행이 직접 움직인다.
- 커뮤니티/홈 섹션 정렬은 오버레이를 쓴다.
- 접근성 보강: 정렬 핸들 `aria-label`, live announcement, 진동 피드백 추가됨.
- 단, 홈 구성 팝업의 섹션 정렬 드래그는 모바일/PWA 환경에서 불안정해 현재는 비활성화했다.

## UI Principles
- 기존 디자인 톤 유지
- 한 번에 전체 리라이트보다 작은 컴포넌트 추출 우선
- 설치 배너/초대 배너/업데이트 배너처럼 상태 충돌이 있는 UI는 우선순위를 명시적으로 관리
- 모바일 전체화면/모달 UI는 하단 네비게이션 높이와 `safe-area-inset`을 기본 전제로 두고, 작은 세로 높이에서도 한 화면에 들어오거나 최소한 자연스럽게 스크롤되게 만든다.
